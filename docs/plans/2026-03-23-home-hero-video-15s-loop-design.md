# 공개 홈 히어로 비디오 15초 루프 자산 재인코딩 설계

## 목표

공개 홈 히어로 배경 비디오를 밝아지는 후반 구간 없이 약 15초 길이의 부드러운 루프로 교체한다. 단순 `playbackRate` 감속이 아니라 원본 자산에서 새 `mp4/webm/poster`를 다시 생성해 프레임 끊김과 루프 경계 튐을 줄인다.

## 현재 상태

- 서비스 자산 [hero-timelapse.mp4](/D:/03_PROJECT/05_mathOCR/04_design_renewal/src/assets/home/hero-timelapse.mp4) 와 [hero-timelapse.webm](/D:/03_PROJECT/05_mathOCR/04_design_renewal/src/assets/home/hero-timelapse.webm) 는 `1280x720 / 20fps / 6.0초` 파생본이다.
- 원본 [star-timelapse.mp4](/D:/03_PROJECT/05_mathOCR/04_new_design/star-timelapse.mp4) 는 `3840x2160 / 25fps / 6.4초` 이고, 밝기 샘플 기준 `4.8초` 이후 급격히 밝아진다.
- 최근 임시 조정은 [PublicHomePage.tsx](/D:/03_PROJECT/05_mathOCR/04_design_renewal/src/app/components/PublicHomePage.tsx) 의 `playbackRate` 와 `timeupdate` 리셋 포인트만 낮춰 체감 주기를 늘린 상태라, 프레임 부족과 밝은 구간 튐을 근본적으로 해결하지 못한다.

## 요구사항 해석

- 밝아지는 구간은 루프 자산에서 아예 제외한다.
- 최종 히어로 비디오는 약 15초로 반복되며, 기존보다 훨씬 덜 끊겨 보여야 한다.
- 기존 poster 기반 정적 폴백은 유지한다.
- 프런트 변경은 새 자산을 참조하는 수준으로 최소화한다.

## 접근 방식

### 권장안

원본 4K 자산에서 어두운 구간만 잘라 기준 시퀀스를 만든 뒤, 프레임 보간으로 길이를 15초까지 확장하고 시작/끝 구간은 짧게 블렌드해 새 루프 자산을 생성한다. 최종 출력은 웹 호환성을 위해 `mp4` 와 `webm`, 정적 폴백용 `poster` 를 함께 갱신한다.

### 대안 1

기존 6초 파생 자산을 그대로 느리게 재생한다. 구현은 가장 쉽지만 이미 줄어든 프레임을 더 오래 늘려 쓰기 때문에 끊김이 그대로 남는다.

### 대안 2

어두운 구간만 잘라 15초로 단순 감속한다. 밝은 구간 제거는 가능하지만 보간이 없어서 사용자가 지적한 낮은 프레임 체감이 남을 가능성이 높다.

## 선택 이유

권장안만이 현재 문제의 두 축인 `밝은 구간 튐` 과 `낮은 프레임 체감` 을 동시에 해결한다. 자산 재생성 비용은 늘지만, 프런트 코드는 거의 건드리지 않고 결과 품질을 직접 통제할 수 있다.

## 아키텍처와 비즈니스 로직 구분

### 아키텍처

- 원본 자산 분석과 자산 재생성 로직을 별도 스크립트로 분리한다.
- 프런트 컴포넌트는 새 자산을 참조하고 브라우저 기본 `loop` 로 재생하게 단순화한다.
- 테스트는 자산 참조와 기본 비디오 속성 계약만 검증하고, 실제 품질 검증은 자산 메타데이터와 브라우저 수동 확인으로 보완한다.

### 비즈니스 로직

- 첫 화면의 브랜드 인상은 어두운 밤하늘 흐름을 유지해야 한다.
- 사용자가 정지나 끊김으로 느끼지 않도록, 루프는 길고 연속적으로 보여야 한다.
- 밝아지는 구간은 인상에 맞지 않으므로 원천 제거한다.

## 자산 생성 원리

1. 원본 비디오 전체 프레임 밝기를 분석해 급격히 밝아지기 직전의 상한 시간을 찾는다.
2. 상한 시간 이전 구간에서 루프 시작/끝 프레임 차이가 가장 적은 지점을 후보로 고른다.
3. 기준 시퀀스를 15초 길이로 확장할 때는 인접 프레임 보간으로 중간 프레임을 생성한다.
4. 마지막 `0.4초` 내외 구간은 첫 구간과 교차 블렌드해 루프 경계를 감춘다.
5. 최종 출력은 `1280x720`, 웹 재생 가능한 `mp4(web faststart, yuv420p)` 와 `webm(vp9)` 로 내보낸다.

## 에러 처리

### 예상 가능한 에러

- `ffmpeg` 또는 `ffprobe` 바이너리 부재
- 원본 비디오 프레임 읽기 실패
- 보간 결과에 심한 고스팅 또는 떨림 발생
- `webm` 인코딩 실패

### 사용자 메시지

- 인코딩 도구가 없으면 “루프 자산 생성 도구가 없어 작업을 완료할 수 없습니다.”
- 원본 분석이나 인코딩이 실패하면 “원본 자산 분석 또는 재인코딩 중 오류가 발생해 새 루프 자산 생성이 중단되었습니다.”
- 품질 기준 미달이면 “자동 보간 결과가 기준 이하라 구간 재선정이 필요합니다.”

## 테스트 전략

- [PublicHomePage.test.tsx](/D:/03_PROJECT/05_mathOCR/04_design_renewal/src/app/components/PublicHomePage.test.tsx) 에서 비디오가 기본 `loop` 로 렌더되고, 수동 `timeupdate` 리셋을 더 이상 쓰지 않는 새 계약을 검증한다.

- 검증 명령:

  - `ffprobe` 로 길이, FPS, 코덱, 해상도 확인
  - `npm run test:run -- src/app/components/PublicHomePage.test.tsx`
  - `npm run build`
- 최종 수동 QA에서는 밝은 플래시 제거, 루프 경계 자연스러움, 브라우저 자동 반복을 확인한다.
